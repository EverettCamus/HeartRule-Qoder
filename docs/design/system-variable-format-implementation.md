# 系统层变量 {%var%} 格式实施方案

**日期**: 2026-02-21  
**版本**: 1.0  
**类型**: 技术实施文档

## 1. 变更概述

### 1.1 目标

实现系统层变量 `{%var%}` 格式支持，与脚本层变量 `{{var}}` 格式严格区分。

### 1.2 核心价值

- **语义清晰**：从符号上区分变量来源（系统 vs 脚本）
- **避免冲突**：系统保留变量不会被脚本变量覆盖
- **作用域明确**：系统变量在所有作用域可见，不受层级限制

### 1.3 变量格式定义

| 格式      | 用途           | 示例                              | 来源                   |
| --------- | -------------- | --------------------------------- | ---------------------- |
| `{%var%}` | 系统层全局变量 | `{%time%}`, `{%who%}`, `{%user%}` | 引擎自动注入           |
| `{{var}}` | 脚本层变量     | `{{咨询师名}}`, `{{用户状态}}`    | 脚本定义或 ai_ask 提取 |

### 1.4 系统保留变量列表

| 变量名 | 说明        | 示例值                |
| ------ | ----------- | --------------------- |
| `time` | 当前时间    | "2026-02-21 09:00:00" |
| `who`  | AI 角色名称 | "AI咨询师"            |
| `user` | 用户称呼    | "来访者"              |

## 2. 实施计划

### 2.1 第一阶段：核心代码修改

#### 文件：`packages/core-engine/src/domain/actions/base-action.ts`

**修改点 1**: 扩展 `substituteVariables` 方法

```typescript
// 原有逻辑：支持 {{var}}, {var}, ${var}
const variablePattern = /\{\{([^}]+)\}\}|\{([^}]+)\}|\$\{([^}]+)\}/g;

// 新增逻辑：优先处理系统层变量 {%var%}
const systemVarPattern = /\{%([^%]+)%\}/g;
```

**修改点 2**: 添加系统变量解析逻辑

```typescript
// 1. 先替换系统层变量（从 context.systemVariables）
// 2. 再替换脚本层变量（从 scopeResolver）
```

**预期行为**:

- 系统变量优先级最高，不可被脚本变量覆盖
- 系统变量在所有作用域可见
- 如果系统变量不存在，保留原占位符

#### 文件：`packages/core-engine/src/domain/actions/action-context.ts`

**新增字段**: `systemVariables?: Record<string, any>`

### 2.2 第二阶段：系统变量注入

#### 文件：`packages/api-server/src/services/session-manager.ts`

**修改点**: 在 Action 执行前注入系统变量

```typescript
const systemVariables = {
  time: new Date().toISOString(),
  who: session.sessionConfig?.aiRole || 'AI咨询师',
  user: session.sessionConfig?.userRole || '来访者',
};
```

## 4. 潜在影响分析

### 4.1 兼容性影响

- **向后兼容**: 仍支持 `{{var}}` 格式，不影响现有脚本
- **迁移成本**: 需要手动更新自定义模板

### 4.2 性能影响

- 增加一次正则匹配（系统变量），性能影响可忽略
- 预计性能下降 < 1ms

### 4.3 风险点

1. **测试覆盖不全**: 可能遗漏某些使用变量的场景
2. **数据库模板未更新**: 需要确保数据库中的模板也更新
3. **自定义模板兼容**: 用户自定义模板需要手动迁移
