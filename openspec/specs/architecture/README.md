# Architecture Design

## Directory Overview

This directory contains system architecture design documentation:

- **Architecture Decision Records (ADRs)**: Key architecture decisions and rationale
- **System Architecture Diagrams**: System components and relationships
- **Technology Selection**: Technology stack evaluation and choices
- **Deployment Architecture**: Deployment environments and configurations

## Language Strategy

- **Primary Language**: English
- **Goal**: Maintain consistency with international technical community, facilitate technical reviews

## File Naming Convention

- `adr-{number}-{brief-description}.md` - Architecture decision record
- `system-architecture.md` - System architecture overview
- `deployment-{environment}.md` - Deployment architecture document
- `technology-{domain}.md` - Technology selection document

## Update Principles

1. Architecture decisions must record context, options, decision, and consequences
2. Architecture diagrams must be kept current, reflecting actual system state
3. Technology selections must be periodically reviewed and updated
