# 开发流程 (Development Process)

## 目录说明

此目录包含团队开发流程和协作规范：

- **Scrum流程**: Sprint规划、执行、评审、回顾流程
- **OpenSpec工作流**: OpenSpec工具使用流程和规范
- **代码质量**: 代码审查、测试、部署流程
- **团队协作**: 沟通、文档、知识管理规范

## 语言策略

- 主要语言：中文
- 目标：确保团队所有成员清晰理解流程

## 文件命名规范

- `scrum-process.md` - Scrum开发流程
- `openspec-workflow.md` - OpenSpec工作流指南
- `code-review-guidelines.md` - 代码审查规范
- `team-collaboration.md` - 团队协作指南

## 流程原则

1. 流程需简单、明确、可执行
2. 定期评审和优化流程
3. 新成员能快速上手
4. 流程支持团队高效协作
